Phase 0 · Docker Review

Docker Container Lifecycle

A recorded terminal session working through Docker end to end — pulling images, running and inspecting containers, port mapping, custom networks, volumes, and building a containerized FastAPI service that runs as a non-root user.

Kyle Miskell — full-stack engineer moving into application security · study plan

01The session

Recorded with asciinema — the text is real, so you can pause and copy any command straight out of the playback. Raw file: docker-lifecycle.cast

02What it covers

  • Container lifecycle — run, stop, pause, unpause, kill, remove, and the difference between docker attach and docker exec
  • Image management — pulling by tag and by organization namespace, tagging, history, inspection, and pruning
  • Runtime inspection — top, stats, inspect, and logs against a running container
  • Networking — publishing ports to the host, creating a custom bridge network, connecting and disconnecting containers
  • Volumes — named and anonymous volumes, persistence across container removal, pruning what is unused
  • Building and serving — a Dockerfile for a Uvicorn-served FastAPI app, run as a non-root user, then the same service under Docker Compose with live reload

The session starts from a deliberately clean slate — a destructive reset gated behind a typed confirmation — so every result is reproducible rather than dependent on leftover state.

04Run it yourself

The script is interactive and pauses between every step, so you can follow along at your own pace.

git clone https://github.com/KLabWeb/cybersecurity-project-00-04-docker-review
cd cybersecurity-project-00-04-docker-review
./containerize-practice-executable.sh

Be aware the script resets your local Docker state before it begins — it removes all containers, images, and volumes on the machine, behind a typed confirmation prompt. It should thus only be run in a disposable test environment.
